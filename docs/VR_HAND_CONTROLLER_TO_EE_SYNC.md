# VR 手柄到 r1pro 控制流程

这份文档只说明当前实际流程和控制映射，不展开实现细节。

完整手柄键位表见 [QUEST3_VR_CONTROL_TABLE.md](QUEST3_VR_CONTROL_TABLE.md)，键盘键位见
[KEYBOARD_CONTROL_TABLE.md](KEYBOARD_CONTROL_TABLE.md)。

## 当前目标

在原先已经跑通的 `earth2moon-sim` 场景里，让 Quest3 手柄直接控制 `r1pro` 或 `ranger_arm`：

- 左右末端位姿
- 左右夹爪开合
- 底盘前后和转向
- 腰部前两个关节

## 当前流程

1. 启动 CloudXR / OpenXR，让 Quest3 和 Isaac Sim 先连上。
2. 进入原场景，保证空间站和 `r1pro` 都在。
3. `keyboard_teleop_ranger_arm.py` 每帧读取左右手柄位姿和按键。
4. 双手柄先做一次标定，再转成左右末端目标位姿。
5. `r1pro` 复用现有控制器 / IK 路径，把末端目标落到双臂。
6. 底盘、腰部、夹爪走独立按键 / 摇杆通道，不和末端位姿混在一起。

## 当前手柄映射

- 同时按住左手柄 `Y` 和右手柄 `B`：做一次 VR 标定。
- 按住 `trigger`：夹爪闭合。
- 松开 `trigger`：夹爪自动张开。
- `squeeze`：保留，不参与机械臂控制，避免握持手柄时误暂停。
- 左手柄摇杆前后：底盘前进 / 后退。
- 左手柄摇杆左右：底盘左移 / 右移。
- 右手柄摇杆左右：底盘原地自旋。
- 左手柄 `X / Y`：腰部关节 1 负向 / 正向。
- 右手柄 `A / B`：腰部关节 2 负向 / 正向。
- 左右手柄 6DoF 位姿：分别控制当前所选机器人左右末端位姿。
- `F3`：同时切换到 `ranger_arm` 头部视角和双臂手柄遥操作，并清除旧标定。
- `F4`：同时切换到 `r1pro` 头部视角和双臂手柄遥操作，并清除旧标定。

说明：

- 这版使用 `trigger` 控夹爪；`squeeze` 暂不绑定功能，避免正常握持造成机械臂停住。
- 底盘改成更接近官方文档：左摇杆负责前后和平移，右摇杆负责自旋。
- 腰部关节改到按键后，不再占用摇杆。
- 标定改成“双手同时按”，避免和日常腰部操作冲突。
- 底盘移动时，双臂目标会继续相对 `r1pro` 机体保持，不应再被世界坐标拖偏。
- 按住夹爪时不再冻结该侧末端目标，夹爪开合和末端位姿跟随分离处理。
- 手柄使用不受 XR 相机锚点影响的物理追踪 pose，并固定转换坐标轴：OpenXR 的 `-Z` 前向映射为 `r1pro` 的 `+X` 前向，OpenXR 的 `+Y` 上方映射为机器人 `+Z` 上方。
- 标定后分别计算手柄相对零位的平移和旋转，再叠加到当时的末端位姿。手柄原地旋转不会额外拉动末端位置。
- 手柄基础位移增益为 `1.6`；前后和上下方向再放大 `1.5` 倍，总增益约为 `2.4`，横向总增益为 `1.6`。末端目标最大平移速度提高到 `2.5 m/s`。
- 末端目标带有速度限制和肩部可达域限制，追踪瞬时跳变或不可达目标不会把双臂拉向头顶。
- IK 对位置和姿态分别限幅，并降低姿态误差权重；优先保证末端位置跟随，同时保留手腕旋转控制。

## 与官方 / 参考实现对齐情况

- 官方 Galaxea 文档里，双臂模式下“手柄移动直接带动末端移动”，`T` 键控制夹爪、`G` 键控制单臂暂停，左摇杆控制前后和平移，右摇杆控制自旋。
- Galaxea 仿真示例采用“目标位姿 -> 机器人控制器跟随”的结构；当前实现同样让 OpenXR 只生成双腕目标，R1 Pro 控制器负责 IK 和关节驱动。
- 不再依赖官方网页、ROS raw pose 或 UDP 桥接，Quest3 直接通过 Isaac Sim OpenXR 输入原场景。

## 视角限制

- 头显视角不再允许无限漂移。
- 当前做法是记录一次标定后的头显参考位姿。
- 当前更偏向“固定在 `r1pro` 头部锚点上，只允许转视角”，不会因为真实头部平移把视角带离机器人头部。

当前优先使用 `r1pro/zed_link/teleop_head_top/Camera`。场景实例未包含该 Camera 时，会创建专用头部 Camera，并在每帧严格复制 `r1pro/zed_link` 的世界位置和朝向；通过 XR profile 将物理头显持续锚定到该视点。机器人平移或自旋时视角同步跟随，真实头部平移不会把视点带离机器人。

`ranger_arm` 头部 Camera 当前朝车体 `+X` 正面，并同时向车头前方、上方各抬一段，优先避开车体中间横梁和内部结构；它会逐帧跟随小车平移和转向。手柄到机械臂的坐标映射保持当前已验证方向，不随本次相机翻转修改。

Ranger 底盘只使用左手柄摇杆：上下为前进/后退，左右为左转/右转；右手柄摇杆不参与 Ranger 底盘控制。

## 当前状态

- 已经接在原先能正常起场景的 `scripts/keyboard_teleop_ranger_arm.py` 里。
- 不再走空场景，空间站和 `r1pro` 沿用原工程。
- `r1pro` 双臂末端、夹爪、底盘、腰部都已具备手柄接入口。
- 手柄旋转已经接入末端姿态控制，不是只控位置。
- 当前保留较快的位置跟随，并对末端平移和旋转做速度限制，让机械夹爪朝向更稳。

## 当前场景初始预设

- `r1pro` 默认放在 `ranger_arm` 正前方，朝向 `180°`，用于面对面摆放。
- 启动时会给 `r1pro` 一个 `arms_forward_level` 初始关节预设。
- 这个预设不再使用冗余 IK 猜测肘部姿态，而是明确设置左右臂第四关节为 `-90°`：上臂竖直向下、小臂水平向前；随后张开夹爪并同步 IK 零位。

  --ground-z -10.0 \
  --r1pro-z 0.0

## 进入场景后的顺序

1. Quest3 连上 CloudXR。
2. Isaac Sim 进入场景后保持整体场景视角，不会自动切到机器人头部。
3. 按 `F3` 选择 `ranger_arm`，或按 `F4` 选择 `r1pro`。
4. 先把双手柄摆到你希望对应当前机器人双臂的起始姿态。
5. 双手同时按一次左手柄 `Y` 和右手柄 `B` 完成标定。
6. 再开始移动手柄、按夹爪、按腰部按键、推底盘摇杆。

标定含义：

- 标定时会以当前所选机器人双末端的真实位姿作为手柄零位。
- 每一侧保存手柄和末端各自的机器人局部零位；后续只使用手柄相对零位变化，不累计世界坐标误差。
